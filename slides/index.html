Two Formats, Two Functions

PyTorch ExportedProgram

% MATLAB net = loadPyTorchExportedProgram("model.pt2"); out = net(input);
  • Full computation graph + weights
  • Exported via torch.export
  • Supports stateful models

LiteRT (TensorFlow Lite)

% MATLAB model = loadLiteRTModel("model.tflite"); out = invoke(model, input);
  • Optimized for edge & mobile
  • Supports int8/float16 quantization
  • Wide ecosystem (Android, MCUs)

What Happens in Colab

  • RepViT — Load pretrained model from timm, export with torch.export, save as .pt2
  • YOLOv11 — Download pre-exported .tflite from GitHub (conversion requires TensorFlow)
  • Stateful LSTM — Define model with explicit state I/O, train on synthetic data, export to .pt2
  • Download all three files to your machine for use in MATLAB
The key API:   torch.export.export(model, inputs)   +   torch.export.save(ep, "file.pt2")
6

The Export Pattern (3 Lines)

Python
# 1. Load or define your model model = timm.create_model('repvit_m1_0', pretrained=True) # 2. Export the computation graph ep = torch.export.export(model, (sample_input,)) # 3. Save to .pt2 torch.export.save(ep, 'repvit.pt2')
MATLAB
% Load and use — that's it net = loadPyTorchExportedProgram("repvit.pt2"); scores = net(input);

Exporting to LiteRT (.tflite)

Option A: ai_edge_torch

Convert any PyTorch model directly to TFLite

Python
import ai_edge_torch model = ... # any PyTorch nn.Module sample = torch.randn(1, 3, 224, 224) edge_model = ai_edge_torch.convert( model, (sample,) ) edge_model.export("model.tflite")

Option B: Framework Export

Some frameworks have built-in TFLite export

Python (Ultralytics)
from ultralytics import YOLO model = YOLO('yolo11s-seg.pt') model.export(format='tflite')

Internally: PyTorch → ONNX → TF → TFLite

MATLAB
% Same MATLAB API regardless of how the model was exported model = loadLiteRTModel("model.tflite"); out = invoke(model, input);

Stateful Models: Explicit State I/O

Python — Model Definition
def forward(self, x, h_0, c_0): out, (h_n, c_n) = self.lstm(x, (h_0, c_0)) pred = self.fc(out) return pred, h_n, c_n
MATLAB — Streaming Inference
[pred, h_n, c_n] = net(x, h_0, c_0); % Next timestep: feed state back [pred, h_n, c_n] = net(x_next, h_n, c_n);
The caller manages the hidden state — essential for Simulink and streaming inference.

Bonus: Verify from MATLAB Using Python

MATLAB
% Load the same .pt2 in Python — directly from MATLAB torch = py.importlib.import_module("torch"); ep = torch.export.load("repvit.pt2"); model = ep.module(); % Run Python inference np = py.importlib.import_module("numpy"); t_in = torch.from_numpy(np.array(input, dtype="float32")); py_out = single(model(t_in).detach().numpy()); % Compare max_err = max(abs(ml_out - py_out), [], 'all');

Key Takeaways

  • Direct codegen — no conversion to MATLAB network objects required (R2026a)
  • PyTorch .pt2 — use torch.export in Python, loadPyTorchExportedProgram in MATLAB
  • LiteRT .tflite — use loadLiteRTModel in MATLAB for edge deployment
  • Stateful models — make hidden state explicit for Simulink integration
  • Python interop — call py.torch from MATLAB for numeric verification
